**Acknowledgement target:** 7 days.<br/>
**Patch target:** depends on severity.

## In scope

Anything that lets an attacker:

- Read or exfiltrate data outside the agent's sandbox (`~/.browy/data/`)
- Execute arbitrary code on the user's machine via prompt injection
  past whatever tools the user has enabled in Settings
- Persist state on the user's machine that survives uninstall
- Connect to a host other than the locally-registered native messaging
  host

## Out of scope

<Aside type="note" title="Prompt injection on visited pages">
Any prompt-injection attack on a visited page can affect the agent's
behavior. That's an inherent property of all page-driving agents, and
the threat model assumes it. Browy bounds the blast radius two ways:
host shell, filesystem, and arbitrary HTTP (`bash`, `read_file`,
`write_file`, `grep`, `glob`, `web_fetch`) are **off by default** and
gated behind a strict allowlist that only honours per-tool opt-ins
from [Settings → Tools](/docs/tools/); and every browser tool that
*is* exposed can be individually disabled in the same place.
</Aside>

Issues in third-party software Browy depends on (Chrome, Node.js,
the GitHub Copilot CLI) — please report those upstream.

## What we do

  <Card title="Patch + ship" icon="rocket">
    A security release goes out as soon as the fix lands.

  <Card title="Add a regression test" icon="approve-check">

## Defense-in-depth notes

For users:

<CardGrid>
  <Card title="Review what's enabled" icon="setting">
    **Settings → Tools** is the source of truth for which browser tools
    the agent can call. Turn off `evaluate_js` or `screenshot` for
    stricter setups.
  </Card>
  <Card title="Host tools are off by default" icon="warning">
    `bash`, `read_file`, `write_file`, `grep`, `glob`, `web_fetch` reach
    your machine — not just the browser tab. They're gated behind
    explicit per-tool opt-in under Settings → Tools → host (advanced).
    Leave them off unless you trust every site you'll visit during the
    session.
  </Card>
  <Card title="Don't run untrusted JavaScript" icon="warning">
    Don't ask the agent to evaluate code (via `evaluate_js` or
    `run_script`) from a source you don't trust.
  </Card>
  <Card title="Trust the yellow banner" icon="warning">
    Chrome's *"Browy started debugging…"* banner is the reminder that
    a session is live. If you see it when you don't expect to,
    disconnect.
  </Card>
</CardGrid>
