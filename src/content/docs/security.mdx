<Aside type="caution" title="Please don't open a public issue for security bugs">
Report privately via
[GitHub Security Advisories](https://github.com/BrowyHQ/browy/security/advisories/new).
This sends a private message to the maintainers and lets us coordinate
a fix and disclosure.
</Aside>

<CardGrid>

</CardGrid>

**Acknowledgement target:** 7 days.<br/>
**Patch target:** depends on severity.

## In scope

Anything that lets an attacker:

- Read or exfiltrate data outside the agent's sandbox (`~/.browy/data/`)
- Execute arbitrary code on the user's machine via prompt injection
  past whatever tools the user has enabled in Settings
- Persist state on the user's machine that survives uninstall
- Connect to a host other than the locally-registered native messaging
  host

## Out of scope

<Aside type="note" title="Prompt injection on visited pages">
Any prompt-injection attack on a visited page can affect the agent's
behavior — that's an inherent property of all page-driving agents, and
the threat model assumes it. The mitigation is **defaults that suit the
task**: powerful tools like host shell and arbitrary HTTP are
[off by default and toggled per-user in Settings](/docs/tools/), so the
blast radius of a hostile page is bounded by what *you* enabled.
</Aside>

Issues in third-party software Browy depends on (Playwright, Chrome,
the GitHub Copilot CLI) — please report those upstream.

## What we do

<CardGrid>
  <Card title="Patch + ship" icon="rocket">
    A security release goes out as soon as the fix lands.
  </Card>
  <Card title="Add a regression test" icon="approve-check">
    If the issue is reproducible, it gets a test so it can't come back.
  </Card>
  <Card title="Credit the reporter" icon="seti:github">
    Acknowledged in the [changelog](/docs/changelog/) (with permission).
  </Card>
  <Card title="Backport if severe" icon="warning">
    If the previous minor is still in wide use and the issue is severe,
    we backport.
  </Card>
</CardGrid>

## Defense-in-depth notes

For users:

<CardGrid>
  <Card title="Review what's enabled" icon="setting">
    **Settings → Tools** is the source of truth for what the agent can
    do on your machine. Turn host shell / filesystem off if you don't
    need them today.
  </Card>
  <Card title="Don't run untrusted scripts" icon="warning">
    Don't ask the agent to `run_script` or run `bash` with code from a
    source you don't trust.
  </Card>
  <Card title="Trust the yellow banner" icon="warning">
    Chrome's *"Browy started debugging…"* banner is your reminder that
    a session is live. If you see it when you don't expect to,
    disconnect.
  </Card>
</CardGrid>
