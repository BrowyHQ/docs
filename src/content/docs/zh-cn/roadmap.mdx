## 接下来要做的事

我们想做的下一批东西，非详尽列表。每一项都对应一个跟踪用的 issue，
你可以点 👍 让它往前排。

- **智能体与人协作的 REPL**：DevTools 面板里的结对编程模式，智能体
  和你轮流编辑同一个文件（Aider 风格）。
- **接入正在运行的会话**：打开一个调试视图来查看和回放正在进行的
  Browy 会话。
- **macOS 与 Linux 安装脚本**：当前只有 Windows 的 PowerShell 一键
  脚本，本地主机本身在三个平台都能跑，只是没有对应的包装脚本。
- **本地模型分支**：智能体循环和 LLM 客户端是解耦的，把 Copilot 换
  成本地 Ollama 风格的模型是可行的。
- **配方 / 保存的工作流**：把一段对话变成一个命名的、可重用的流程
  （"每周一总结一下我的 Gmail 未读"）。
- **按站点划分的记忆空间**：把 `remember` 工具按来源域名隔离，
  避免偏好在站点间泄漏。
- **多标签页协作工具**：超出 `tabs.list` / `switch_tab` 的能力，
  比如"打开这 5 个 URL，从每一个里抓 X"。
- **基于截图的兜底验证**：针对 canvas / WebGL 等 DOM diff 无可读
  内容的界面。
