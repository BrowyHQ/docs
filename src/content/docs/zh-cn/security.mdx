## 在范围内

任何能让攻击者做到以下事情的问题：

- 读取或导出智能体沙箱目录（`~/.browy/data/`）之外的数据
- 通过提示词注入绕过用户在设置里启用的工具，在用户机器上执行任意代码
- 在用户机器上留下卸载后仍然存在的状态
- 连接到本地注册的原生消息主机之外的主机

## 不在范围内

<Aside type="note" title="访问页面上的提示词注入">
任何被访问页面上的提示词注入都可能引导智能体。这对所有驱动页面的
智能体都是如此，威胁模型已经假设了这一点。Browy 默认关闭所有触碰
主机的工具（`bash`、`read_file`、`write_file`、`grep`、`glob`、
`web_fetch`），需要时在 [设置 → 工具](/zh-cn/tools/)里按工具开启，
每个浏览器工具也都可以单独关闭。
</Aside>

第三方依赖（Chrome、Node.js、GitHub Copilot CLI）里的问题请上报到
对应项目。

## 纵深防御说明

针对用户：

<CardGrid>
  <Card title="检查启用了哪些工具" icon="setting">
    **设置 → 工具**是智能体可以调用哪些浏览器工具的唯一来源。
    在严格的场景下可以关掉 `evaluate_js` 或 `screenshot`。
  </Card>
  <Card title="主机端工具默认关闭" icon="warning">
    `bash`、`read_file`、`write_file`、`grep`、`glob`、`web_fetch`
    访问的是你的机器，而不是浏览器标签页。需要时在
    设置 → 工具 → 主机（进阶）按工具开启。
  </Card>
  <Card title="不要执行不可信的 JavaScript" icon="warning">
    不要让智能体（通过 `evaluate_js` 或 `run_script`）执行你不信任
    来源的代码。
  </Card>
  <Card title="留意调试横幅" icon="warning">
    *"Browy started debugging..."* 横幅表示一个会话正在进行。
    如果不是你启动的，断开连接。
  </Card>
</CardGrid>
