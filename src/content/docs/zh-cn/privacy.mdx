Browy 是一个桌面应用：一个基于 Chromium 的浏览器扩展，加一个跑在
你机器上的小型原生消息主机。这一页说明 Browy 处理什么数据，也是
Chrome Web Store 和 Microsoft Edge Add-ons 商店页面引用的隐私政策。

## Browy 把什么发出去，发给谁

<Tabs>
<TabItem label="发给 GitHub Copilot">

| 数据 | 为什么 |
|---|---|
| 智能体观察到的页面内容（DOM 快照、截图、控制台 / 网络日志、JS 执行结果） | 模型需要看到屏幕上的内容才能行动。和在终端里用 `gh copilot` 走的是同一条数据通道。 |
| 你的对话消息 | 这些是给模型的 prompt。 |
| 工具调用结果（比如 "navigated to X"、"clicked element 7"） | 模型需要知道结果才能规划下一步。 |

GitHub 对 Copilot 数据的处理由
[GitHub Copilot 隐私声明](https://github.com/customer-terms/github-copilot-product-specific-terms)
和你自己的 Copilot 订阅条款约束。Browy 不修改、拦截或代理这些请求。

</TabItem>
<TabItem label="发给 BrowyHQ">

BrowyHQ 没有任何后端服务。扩展和原生主机都不会向 BrowyHQ 发送
任何分析、遥测、错误报告或使用数据。

</TabItem>
<TabItem label="发给其他任何人">

扩展不加载第三方脚本，也不调用任何第三方服务，除了本地的 Copilot CLI。

</TabItem>
</Tabs>

## Browy 在本地存储什么

| 数据 | 位置 | 生命周期 |
|---|---|---|
| 对话历史 | `chrome.storage.local`（按 profile 隔离的扩展存储） | 在侧边栏里清掉或者卸载扩展为止 |
| 设置（模型、推理强度、主题、上次使用的标签页） | `chrome.storage.local` | 同上 |
| 原生主机日志 | `~/.browy/host/host.log`（单文件循环，5 MB） | 循环时被覆盖；卸载时删除 |
| 智能体临时磁盘 + 键值记忆 | `~/.browy/data/files/` 和 `~/.browy/data/notes.json` | 你删为止；跨对话保留 |
| GitHub Copilot 会话状态 | `~/.copilot/`（由 Copilot CLI 管理，不由 Browy 管理） | 看 Copilot CLI 自己的行为 |

## 由你决定要分享什么

智能体读取的页面内容，包括 DOM 快照、控制台输出、网络日志、截图，
来自当前激活的标签页，原封不动地发给模型。你决定智能体作用在哪个
标签页、对话里输入什么、设置里启用了哪些工具。

这里也包括高级用户*主动*想发送的东西：后端测试用的 API key、OAuth
重定向 URL、网络抓包里的 `Authorization` 头。Browy 不替你二次猜疑。

<Aside type="tip" title="默认值">
浏览器工具默认开启；触碰主机的工具（`bash`、`web_fetch`、`read_file`、
`write_file`、`grep`、`glob`）默认关闭，需要在
**设置 → 工具**里启用。两边都可以按工具粒度开关。
</Aside>

## 儿童

Browy 不面向 13 岁以下儿童，也不会有意识地收集任何人（包括儿童）的
任何数据。

## 政策变更

这份政策变更时，页面顶部的*最近更新*徽章也会变。重要变更会同时
记录在[更新日志](/zh-cn/changelog/)中。
