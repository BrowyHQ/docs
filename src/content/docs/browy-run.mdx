`browy run` is the headless side of Browy: a terminal agent powered by
your existing GitHub Copilot subscription, with the SDK's full default
toolset (`read_file`, `write_file`, `bash`, `grep`, `glob`,
`web_fetch`). No browser required.

Think of it as a `gh copilot`-style CLI scoped to your current
directory. Useful for quick refactors, summarising a diff, sweeping
TODOs, or driving a build/test loop.

<Aside type="note">
`browy run` and Browy's browser surfaces (side panel, DevTools panel)
are independent. The browser surfaces speak browser tools only;
`browy run` speaks host tools only. They never mix toolsets, so a
prompt-injected page can't reach your filesystem.
</Aside>

## One-shot tasks

```bash
browy run "summarize the staged diff in one paragraph"
browy run "find every TODO in src/ grouped by file"
browy run "look up the README for the express package on npm"
```

The reply streams to stdout; the process exits when the agent is done.

## Interactive REPL

```bash
browy run
# browy> refactor utils.ts to extract the date helpers into date-utils.ts
# (agent works…)
# browy> now add unit tests for the new file
# browy> exit
```

Type `exit` (or hit `Ctrl+D`) to quit.

## Resume a session

```bash
browy run --list                       # show recent sessions
browy run --resume sess-abc123         # pick up where you left off
```

Sessions live under `~/.browy/cli-sessions/`. They're isolated from
both Browy's browser sessions and your own `copilot` CLI sessions, so
you never see them in `copilot --list` and vice versa.

## Flags

| Flag | What it does |
|---|---|
| `--model <id>` | Override the model (e.g. `gpt-5-mini`, `claude-sonnet-4.5`). |
| `--cwd <path>` | Working directory for file/shell tools. Default: `process.cwd()`. |
| `--resume <id>` | Resume a prior session by id. |
| `--list` | List recent `browy run` sessions and exit. |
| `--help` | Show help. |

## Available tools

| Tool | What it does |
|---|---|
| `read_file` | Read a file from disk. |
| `write_file` | Create or overwrite a file. |
| `bash` | Run a shell command and capture stdout/stderr. |
| `grep` | Ripgrep-style code search. |
| `glob` | Find files by glob pattern. |
| `web_fetch` | Fetch a URL and return text. |

These are the GitHub Copilot SDK's defaults — the same set the
`copilot` CLI exposes. Browy doesn't filter them in `browy run`
because the use case _is_ "I want a terminal coding agent".

<Aside type="caution">
`bash` is real shell access. Treat `browy run` like any other
coding-agent CLI: point it at code you trust, and review file edits
and shell commands before letting them run.
</Aside>

  <Card title="Quick repo work" icon="seti:git">
