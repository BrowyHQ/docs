Browy is two pieces:

1. **The native host** — a small Node.js helper that runs on your machine
   and talks to GitHub Copilot.
2. **The browser extension** — a Chrome side panel that talks to the
   native host.

The installer below sets up both.

<Aside type="caution" title="Early access">
Today the only supported install channel is the GitHub release. The
Chrome Web Store listing is in review — once approved, you'll be able
to install with one click.
</Aside>

## Step 1 — install the native host

<Tabs>
<TabItem label="Windows" icon="seti:windows">

Open PowerShell and run:

```powershell
irm https://github.com/BrowyHQ/browy/releases/latest/download/install.ps1 | iex
```

The installer will:

<Steps>
1. Download the latest Browy release (~130 MB)
2. Install the native host to `%LOCALAPPDATA%\Browy\app\`
3. Register `com.browy.host` with Chrome / Edge / Brave
4. Copy the unpacked extension to `%USERPROFILE%\Downloads\Browy-Extension\`
5. Open the extension folder in Explorer when it's done
</Steps>

## Step 2 — load the extension

<figure class="browy-frame wide">
  <Image src={placeholder} alt="chrome://extensions/ with the Browy extension loaded and pinned" />
  <figcaption>chrome://extensions with Developer mode on and the Browy folder loaded.</figcaption>
</figure>

<Steps>
1. Open `chrome://extensions/` (or `edge://extensions/`, `brave://extensions/`)
2. Toggle **Developer mode** on (top-right of the page)
3. Click **Load unpacked**
4. Select `~/Downloads/Browy-Extension/`
5. Pin the Browy icon to your toolbar so the side panel is one click away
</Steps>

<Aside type="tip" title="Why developer mode?">
The Chrome Web Store listing is still in review, so the extension isn't
signed by Google yet and has to load as unpacked. Developer mode is the
toggle that allows that. Once Browy ships through the Web Store, this
step goes away.
</Aside>

## Step 3 — sign in to Copilot

Click the Browy icon to open the side panel. The first time you use it,
Browy will prompt you to sign in.

<figure class="browy-frame">
  <Image src={panelSignin} alt="The Browy side panel with the 'Sign in to GitHub Copilot' button visible" />
  <figcaption>Click "Sign in to GitHub Copilot" and follow the device-flow link.</figcaption>
</figure>

A terminal window opens with the GitHub Copilot **device-flow link**.
Paste the code into the URL it shows, authorize Browy in your browser,
and the terminal closes itself.

You're done. Try [your first chat →](/docs/first-chat/)

## Updating

Re-run the installer. It detects your existing install and updates in
place.

```powershell
irm https://github.com/BrowyHQ/browy/releases/latest/download/install.ps1 | iex
```

Then **reload** the extension in `chrome://extensions/` (the circular ↻
icon on the Browy card).

## Uninstall

<Steps>
1. Remove the extension from `chrome://extensions/`
2. Delete `%LOCALAPPDATA%\Browy\` and `%USERPROFILE%\.browy\`
3. Delete the native messaging host registration:
   `%LOCALAPPDATA%\Google\Chrome\User Data\NativeMessagingHosts\com.browy.host.json`
   (and the equivalent under Edge / Brave folders)
</Steps>

A scripted uninstaller is on [the roadmap](/docs/roadmap/).

## Troubleshooting

<Aside type="caution" title="“Host disconnected — reconnecting…”">
The side panel can't reach the native host. The most common causes are:
the native host failed to install (re-run the installer), Chrome was
opened before the host was registered (restart Chrome), or your
antivirus quarantined `node.exe` (allow `%LOCALAPPDATA%\Browy\` in your
AV exclusions).
</Aside>

<Aside type="caution" title="“Host unreachable — reinstall →”">
The reconnect attempts gave up. Click the **Reinstall host** link in
the banner — it points at the same `install.ps1` you used the first
time. Your chats and settings are preserved across reinstalls.
</Aside>

<Aside type="caution" title="“Browy started debugging this browser”">
This is Chrome's standard yellow banner — it's non-removable and shown
whenever any extension uses `chrome.debugger`. It's how Browy
sees and acts on the page; closing it disconnects the agent. See
[Permissions → debugger](/docs/permissions/) for why this is required.
</Aside>
