Browy is a desktop application: a Chromium browser extension plus a
small native messaging host that runs on your machine.

<Aside type="tip" title="Summary">
**BrowyHQ runs no servers.** The Browy organization does not receive,
store, log, or analyze any data about your usage. LLM traffic flows
between your machine and **GitHub Copilot** through the locally-installed
[GitHub Copilot CLI](https://docs.github.com/copilot/concepts/agents/about-copilot-cli).
</Aside>

This page is the privacy policy referenced in the Browy Chrome Web Store
and Microsoft Edge Add-ons listings.

## What Browy sends, and to whom

<Tabs>
<TabItem label="To GitHub Copilot">

| Data | Why |
|---|---|
| Page content the agent inspects (DOM snapshots, screenshots, console / network logs, JS evaluation results) | The model needs to see what's on screen to act. Same data path as `gh copilot` in a terminal. |
| Your chat messages | They're the prompt to the model. |
| Tool-call results (e.g. "navigated to X", "clicked element 7") | The model needs feedback to plan the next step. |

GitHub's handling of Copilot data is governed by the
[GitHub Copilot Privacy Statement](https://github.com/customer-terms/github-copilot-product-specific-terms)
and your Copilot subscription terms. Browy does not modify, intercept,
or proxy these requests.

</TabItem>
<TabItem label="To BrowyHQ">

| Data | What we receive |
|---|---|
| Telemetry | Nothing. |
| Analytics | Nothing. |
| Error reports | Nothing. |
| Usage stats | Nothing. |

There is no analytics SDK in the extension or the host, and no Browy
backend to receive any of it.

</TabItem>
<TabItem label="To anyone else">

Nothing. Browy does not call any third-party service except the local
Copilot CLI. The extension does not load any third-party scripts.

</TabItem>
</Tabs>

## What Browy stores locally

| Data | Where | Lifetime |
|---|---|---|
| Chat history | `chrome.storage.local` (per-profile extension storage) | Until you clear it from the side panel or uninstall |
| Settings (model, reasoning effort, theme, last tab) | `chrome.storage.local` | Same |
| Native host log | `~/.browy/host/host.log` (single rotating 5 MB file) | Overwritten on rotation; deleted on uninstall |
| Agent scratch disk + key-value memory | `~/.browy/data/files/` and `~/.browy/data/notes.json` | Until you delete them; survives across chats |
| GitHub Copilot session state | `~/.copilot/` (managed by the Copilot CLI, not Browy) | Per Copilot CLI's behavior |

<Aside type="note">
Nothing in this list leaves your machine unless you initiate it. Asking
the agent to "open this in a new tab" causes a navigation; asking it to
"save this to my Downloads" calls `chrome.downloads`.
</Aside>

## You decide what to share

Page content the agent reads — DOM snapshots, console output, network
logs, screenshots — all comes from the active tab and is sent to the
model unmodified. You pick the tab the agent runs against, what you type
into chat, and which tools are enabled in Settings.

That includes things power users *want* to send: API keys for backend
testing, OAuth redirect URLs, `Authorization` headers from a network
trace. Browy doesn't second-guess you.

<Aside type="tip" title="A few sensible defaults">
The Copilot SDK's most powerful built-in tools — host shell (`bash`),
arbitrary HTTP (`web_fetch`), host filesystem (`read_file` /
`write_file`) — are off by default. Turn them on per-tool in
**Settings → Tools**. Default-on tools are the browser-driving ones in
[Tools the agent has](/docs/tools/).
</Aside>

## How Browy is built

<CardGrid>
  <Card title="No Browy backend" icon="rocket">
    BrowyHQ runs no servers. Page content goes to GitHub Copilot through
    the local CLI, the same way `gh copilot` works in a terminal.
  </Card>
  <Card title="No background activity" icon="moon">
    The agent acts only when you send a message. There is no background
    crawler, prefetch, or scraper.
  </Card>
  <Card title="No telemetry" icon="seti:lock">
    No analytics SDK in the extension or the host. Updates ship through
    the Web Stores (extension) and GitHub Releases (CLI host).
  </Card>
  <Card title="No third-party scripts" icon="seti:javascript">
    The extension loads zero external scripts. The side panel is fully
    bundled.
  </Card>
</CardGrid>

## Children

Browy is not directed to children under 13, and we do not knowingly
collect any data from anyone, including children.
